Interfaces de Repositório: > "Entendi que a interface no domain funciona como um contrato/manual. Ela define o que o sistema precisa (ex: buscar por CPF), mas deixa o como (SQL, Prisma, API externa) para ser decidido depois na camada de infra. Isso protege meu domínio de mudanças tecnológicas e facilita os testes, pois posso criar um repositório 'fake' na memória para testar minha lógica sem precisar de um banco de dados real."



Termo	   /      O que ele é na vida real?	   /    O que ele faz no código?

Entidade	/  A Ficha da Pessoa	/ Valida se os dados da pessoa estão certos (CPF, Nome).

Repositório	/ O Arquivista(ou Porteiro) /	É o único que toca no banco de dados para buscar ou guardar fichas.


Use Case	/  O Processo de Cadastro  /	É o passo a passo: Checar CPF -> Criar Ficha -> Mandar salvar.



📒 Diário de Bordo: O Papel do Repositório no DDD
O que é o Repositório?
É o Porteiro do Banco de Dados. Ele é o único componente do sistema que tem permissão para tocar nas tabelas do banco e transformar "dados brutos" (linhas e colunas) em "Entidades" (nossas classes ricas, como Person ou Gender).

Por que usamos uma Interface no Domínio?
Criamos o Contrato do Porteiro. Eu defino o que o porteiro deve saber fazer (ex: "buscar por CPF", "salvar ficha"), mas não me preocupo agora com como ele fará isso (se usará uma chave física, digital ou biométrica). Isso protege meu sistema: se eu decidir trocar o banco de dados no futuro, só preciso trocar o "porteiro" (a implementação), sem mexer na lógica de negócio.

Diferença Prática no Fluxo:

Entidade (A Ficha): Valida se os dados estão corretos internamente (Ex: "O nome está vazio?").

Repositório (O Porteiro/Arquivista): É quem busca e guarda as fichas no arquivo (Banco de Dados).

Use Case (O Processo): É o mestre de cerimônias. Ele pede ao Repositório para buscar uma ficha, verifica a regra de negócio e, se estiver tudo ok, manda o Repositório guardar a ficha novamente.

Conclusão do dia:
O Repositório não é o banco de dados e nem o processo de cadastro. Ele é a ponte especializada que permite que o resto do sistema converse com o armazenamento de forma organizada e segura.
